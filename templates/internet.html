The Internet

Networking Vocabulary

Basic Format: protocol://host/path?query
Network: A network is a group of entities that can communicate, even though they are not directly connected.
Latency: The time it takes for a message to get from the source to the destination usually expressed in milliseconds (1/1000th of a second).
Bandwidth: The amount of information that can be transmitted per unit of time usually expressed as Mbps (million bits per second).
Bit: The smallest unit of information expressed as an off/on (0/1) state.
Binary Questions: Questions with a 50/50 chance of having either answer be correct which reduces the size of queries by half each time for the most efficient acquisition of information.
Port: Follows the colon in the address and specifies which program, process, or service is being reached on the server.
Protocol: The set of rules that people agree to that tell you how two entities talk to each other. For the web, client and server communication is handled by the HyperText Transfer Protocol (HTTP).

Queries

Format: name = value
First parameter is separated from the URL by ?
Following parameters are separated by &
Fragments, which are client-side and follow a #, refer to a part of a webpage

An example based on a game's bulletin board:
http://www.somegame.com/viewtopic.php?f=12&t=3456#p7890

"f" is a query for the forum, "t" is a query for the thread," and "#p" is a fragment that automatically moves the page to display a particular post.

Servers

Servers host literally everything that is on the internet. They receive requests from clients (typically a user's web browser) and return either static or dynamic content. Static content doesn't change and is things like pre-written files and images. Dynamic content is made as it is requested and includes web applications.

Requests

Example: GET /foo HTTP/1.1

Method: GET (in this example) or POST
Path: /foo
Protocol/Version: HTTP/1.1

GET:
  • Parameters in URL
  • Used for fetching documents
  • Limited by maximum URL length
  • OK to cache
  • Shouldn't change the server (can make the same request repeatedly)
POST:
  • Parameters in body
  • Used for updating data
  • No max length
  • Not OK to cache
  • OK to change the server (that's the point)

Responses

Example: HTTP/1.1 200 OK

Protocol/Version: HTTP/1.1
Status Code and Reason Phrase:

  • 200 - OK
  • 302 - Found
  • 404 - Not Found
  • 500 - Server Error

Headers

Content-Length: 1539
Content-Type: text/html;
Date: Mon, 08 Feb 2018 20:59:15 GMT
Host: www.example.com (because machines can have many names)
Server: Apache /2.2.3 (Don't include this! Security risk!)
User-Agent: Chrome (usually your browser)

Forms

Forms are POST requests that submit data to servers with the following elements:

<form action="http://www.example.com"> is where the input submits to
<input type="text"> is the default and creates a typeable form
<input type="password"> replaces the input with dots but sends it in insecure plain text
<input type="checkbox"> creates a checkbox
<input type="radio"> creates buttons that are grouped by name where selecting on button deselects the rest
<input name="example"> is the name of the input field
<input value="value"> sets the value of a button
<label> put inputs here to label them </label>
<select>creates a drop-down menu

<form>
<input type="text">
<input type="password">
<input type="checkbox">
</form>
