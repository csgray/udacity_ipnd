Python

Special Notes

Python is case-sensitive.
Python is space-sensitive and will generate an error if there is a space in front of a command.
Python comments start with #
You can break up lines of code with a \

Vocabulary

Computer: A universal machine that can be programmed to do any computation
Expression: Something that has a value.
Program: What tells the computer what steps to take
Precise Squence of Steps: Series of instructions to accomplish a task
High-Level Language: Input that works through an interpreter and not on the computer directly
Input: Programs that we write
Interpreter: Executes our programs on the computer
Method: Procedure built into Python

Natural Languages vs Code

Natural languages are ambiguous: Different people can interpret the same sentence to mean different things, but we want computers to interpret things the same way every time.

Natural languages are verbose: It takes too many words to detail every step of even a simple action.

Example: print 2 + 2 +
We can figure out what this means, but Python produces a syntax error.

Backus-Naur Form

The full Backus-Naur Form is pretty complicated, as this Wikpedia article shows, but the gist is that it use symbols to show how computer programming languages work. The gist of it is that non-terminals (variables) are on the left side with an arrow pointing towards their replacements which can be other variables, the same variable, or terminals which have no replacements and never appear on the left side of the form.

A derivation is when you keep working down the expression until all of the non-terminals are replaced by terminals. We can explain Python that way.

Python Grammar for Arithmetic Expressions

<expression> → <expression> <operator> <expression>
<expression> → <number>
<expression> → ( <expression> )
<operator> → +
<operator> → −
<operator> → *
<operator> → /
<number> → 0, 1, ...

Note: Python ignores the decimal (remainder) when you divide an integer by another integer. You need to give one of the variables a decimal, even a .0, to get decimal results.

Variables

Variables make it easy to refer to expressions because rather than type the whole expression out again you can refer to its variable. It simultaneously labels the expression and lets you refer back to it to reduce the amount of repeat code.

A 'magic number' is a number without a variable. Those are bad.

You assign a value to a variable when you state what expression or number it represents. You do that using = which does not mean "equals" like in math but is an arrow indicating that whatever is to the right of the = is the value of the variable on the left.

Arithmetic: 2 + 3 = 5
Python: my_variable = 5

The above is an example of an Assignment Statement: name = expression

Python will keep calculating and updating the value of a variable as it moves down the code.

days = 7 * 7 # Result: 49
days = 48 # Changes the value to 48
days = days − 1 # Result: 48 - 1 = 47. Variable has a new value.
days = days − 1 # Result: 47 - 1 = 46

Strings

Python isn't just numbers. You can have sequences of characters called strings. Strings are enclosed in single or double quotes, ' or ", to delineate them from variables. It doesn't matter which you use so long as the string begins and ends with the same one.

print 'Hello' # Result: Hello
print "Hello" # Result: Hello

You can add strings together which is called "concatenation", but you cannot concatenate strings and integers. You need to include the spaces in the strings as Python will not add them. You can multiply strings.

name = "Corey"
print "Hello" + name # Result: HelloCorey
print "Hello " + name + 10 * "!" # Result: Hello Corey!!!!!!!!!!

Note that strings are not integers. You cannot do math with strings. Instead of being summed, numbers in strings are printed side by side.

print 2 + 2 # Result: 4
print "2" + "2" # Result: 22

You can select the characters of a string, called 'indexing', by putting an integer or expression in square brackets. The first character in a string is 0 and count right. Negative numbers start at the end of the string with -1 and count left.

name = "Corey"
print name[0] # Result: C
print name[2] # Result: r
print name[-1] # Result: y

Trying to index a character that isn't in the string produces an error:

IndexError: string out of range

It is possible to index more than one character. You can specify a starting and stopping character then use the characters between them. The index will stop immediately before the stopping character. An empty specification will run to the beginning or end of the string.

word = 'assume'
print word[3] # Result: u
print word[3:4] # Result: u
print word[3:3] # This stops where it starts. Result:
print word[4:6] # From 4 to 6. Result: me
print word[4:] # From 4 to end. Result: me
print word[:2] # From beginning to 2. Result: as
print word[:] # From beginning to end. Result: assume

It is also possible to search for a string within a string.

<string1>.find(<string2>)

This will not print the string itself but the numerical position of the first character of the second string within the larger string. The result be -1 if the string can't be found. An empty string, meaning no space between the quotation marks, will always be found. Python thinks that every string begins with the empty string.

print "test".find("te") # Result: 0
print "test".find("st") # Result: 2
print "text".find("text") # Result: 0
print "text".find("Text") # Python is case sensitive. Result: -1
print "text".find("") # This is an empty string. Result: 0
print "text".find(" ") # This is a space. Result: -1

You can find the second, third, and so on occurrence of a string by specifying a starting position after the first character of the string.

<string1>.find(<string2>, <number>)
print "test".find("t") # Result: 0
print "test".find("t", 1) # Result: 3

There is also a built-in method for replacing part of a string with a new string.

<string>.replace(<old>, <new>[, <max>])

The [, <max>]; part is optional and specifies the maximum number of times to make the replacement. Don't include the square brackets as that is just notation for optional.

sentence = "A NOUN went on a walk. They can VERB really well."
sentence = sentence.replace("NOUN", "duck")
sentence = sentence.replace("VERB", "waddle")
print sentence # Result: A duck went on a walk. They can waddle really well.
